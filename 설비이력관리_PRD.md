# 설비 이력관리 웹앱 — PRD (제품 요구사항 정의서)

**버전:** v0.1 (인터뷰 정리 초안) | **작성일:** 2026-06-08 | **상태:** 초안 — 미확정 항목 있음 (8장 참조)

---

## 1. 개요

회사 전체 설비 자료(이력·데이터)를 관리하고, 향후 5년간 정비 계획을 수립하는 업무를 지원하는 도구. 현재 수기(종이) + 엑셀로 관리하면서 발생하는 **데이터 불일치 문제**를 줄이는 것이 1차 목표.

전체 업무 중 **이력관리를 1순위**로 정하고, 5년 정비 계획은 후속 단계로 둔다.

---

## 2. 배경 & 문제 정의

현재 업무 방식과 문제점:

- 설비 자료를 주로 **종이(수기)**로 관리하고, 동시에 **엑셀**에도 입력해 관리한다.
- **문제 ①** 종이 자료와 엑셀 데이터의 값이 서로 다르다.
- **문제 ②** 10개 지사가 입력한 값이 제각각이다(같은 항목을 다르게 적음).
- 기록은 **분기별로 소분하여** 관리하고, **연 1회 전체 데이터를 재확인**한다.

핵심 통찰: "제각각" 문제의 근본 해법은 **모두가 동일한 입력 양식(동일 드롭다운·동일 검증 규칙)을 쓰게 하는 것**이다.

---

## 3. 목표 & 범위

### 3.1 목표

- 10개 지사가 **동일한 표준 양식**으로 설비 이력을 기록하게 한다.
- 종이 ↔ 엑셀 불일치를 추적·대조할 수 있게 한다(근거문서번호 연결).
- 본사가 지사 데이터를 **한 번에 취합·검증·불일치 확인**할 수 있게 한다.

### 3.2 범위 (Phase 1)

- 설비 마스터 관리, 이력 기록, 검색/필터, 표준 파일 내보내기/불러오기, 검증·취합.

### 3.3 범위 밖 (후속 단계)

- 5년 정비 계획 자동 수립 (Phase 2)
- 다수 사용자 실시간 공유 시스템 (Phase 3, IT 협조 필요)

---

## 4. 사용자 & 사용 시나리오

| 사용자 | 역할 | 주요 행동 |
|---|---|---|
| 지사 담당자 (10개 지사) | 입력 주체 | 표준 양식으로 설비·이력 기록, 표준 파일로 내보내 인트라넷에 올림 |
| 본사 담당자 (작성자 본인) | 취합·검증 주체 | 지사 파일들을 불러와 검증·취합, 불일치 확인, 연 1회 전체 재확인 |

**주기:** 분기별 기록 → 연 1회 전체 취합·재확인.

---

## 5. 방식 (아키텍처) 결정

**채택: 방식 A — 표준 입력 + 취합 HTML 도구 (파일 교환 방식)**

- 지사·본사가 **동일한 HTML 도구**를 사용한다.
- 지사는 표준 형식 파일로 내보내 **인트라넷 공유 폴더**에 올린다.
- 본사는 그 파일들을 불러와 검증·취합한다.
- 실시간이 아니라 **주기적 파일 취합** 방식(분기 단위 업무 흐름과 맞음).

**기술적 한계 (확정 사항):** 순수 HTML 파일 하나만으로는 여러 사용자의 데이터가 자동으로 합쳐지지 않는다. 그래서 "파일 내보내기/불러오기"가 핵심 기능이다.

**환경:** 사내 인트라넷/공유 서버 사용 가능.

**향후 확장:** 잘 자리잡으면 IT 협조하에 실시간 공유 웹앱(서버+DB)으로 확장 (Phase 3).

## 6. 데이터 모델

설계 원칙: **불일치가 잘 생기는 항목은 자유입력을 막고 드롭다운으로 고르게 한다.**

### 6.1 설비 마스터 (설비 1대 = 1행)

| 항목 | 입력방식 | 필수 | 설명 |
|---|---|---|---|
| 관리번호 | 자유입력(규칙) | ✱ | 설비 고유 ID, 모든 이력의 연결 열쇠 (예: `SE-PMP-001`) |
| 설비명 | 자유입력 | ✱ | 예: 1호 냉각수펌프 |
| 설비종류 | 드롭다운 | ✱ | 아래 6.3 목록 |
| 지사 | 드롭다운 | ✱ | 10개 지사 중 선택 |
| 세부위치 | 자유입력 | | 동·층·실 |
| 제조사 | 자유입력 | | |
| 모델/규격 | 자유입력 | | |
| 설치일 | 날짜 | | YYYY-MM-DD |
| 상태 | 드롭다운 | ✱ | 사용중 / 예비 / 정비중 / 폐기 |
| 정비주기(개월) | 숫자 | | 5년 계획 자동 생성에 사용 (Phase 2) |
| 비고 | 자유입력 | | |

### 6.2 이력 기록 (점검·정비 1건 = 1행)

| 항목 | 입력방식 | 필수 | 설명 |
|---|---|---|---|
| 기록ID | 자동 | ✱ | 시스템 부여 |
| 관리번호 | 드롭다운(마스터 연동) | ✱ | 이름이 아니라 번호로 설비 연결 |
| 일자 | 날짜 | ✱ | YYYY-MM-DD |
| 분기 | 자동 | | 일자에서 자동 계산 (예: 2026-Q1) |
| 작업유형 | 드롭다운 | ✱ | 정기점검 / 예방정비 / 고장수리 / 부품교체 / 개선 |
| 작업내용 | 자유입력 | ✱ | |
| 결과/판정 | 드롭다운 | ✱ | 정상 / 이상-경미 / 이상-중대 / 조치완료 / 조치필요 |
| 담당자 | 자유입력 | | (드롭다운 전환 여부 미정 — 8장) |
| 부품/교체내역 | 자유입력 | | |
| 비용(원) | 숫자 | | 단위·부가세 기준 통일 필요 (8장) |
| 다음 예정일 | 날짜 | | 비우면 정비주기로 자동 계산 가능 |
| 근거문서번호 | 자유입력 | | 종이 대장 번호 → 종이↔디지털 대조용 |

### 6.3 드롭다운 기본 보기 (확정)

- **설비종류:** 펌프 / 모터 / 공조 / 전기 / 배관 / 계측 / 소방 / 기타
- **작업유형:** 정기점검 / 예방정비 / 고장수리 / 부품교체 / 개선
- **결과/판정:** 정상 / 이상-경미 / 이상-중대 / 조치완료 / 조치필요
- **상태:** 사용중 / 예비 / 정비중 / 폐기

### 6.4 데이터 규칙 (확정)

- **관리번호 규칙:** `[지사코드 2자]-[종류코드 3자]-[일련번호 3자]` (예: `SE-PMP-001`). 규칙 준수 시 10개 지사 번호 미중복 보장.
- **날짜 형식:** YYYY-MM-DD
- **이력↔설비 연결:** 설비명이 아닌 **관리번호**로 연결.

## 7. 기능 요구사항 (Phase 1)

### 7.1 입력 (지사·본사 공통)

- **F-1.** 설비 마스터 등록/수정 (드롭다운·필수값 검증 적용)
- **F-2.** 이력 기록 등록/수정 (관리번호로 설비 선택, 드롭다운·필수값 검증)
- **F-3.** 입력값 검증: 필수 항목 누락, 날짜 형식 오류, 미등록 관리번호 등 입력 시점에 차단/경고
- **F-4.** 분기 자동 계산, 다음 예정일 자동 계산(정비주기 기반)

### 7.2 조회

- **F-5.** 설비/이력 목록 보기
- **F-6.** 검색·필터 (지사, 설비종류, 작업유형, 결과, 기간/분기 등)

### 7.3 데이터 교환 (방식 A의 핵심)

- **F-7. 내보내기:** 표준 형식 파일로 출력 (지사 → 인트라넷)
- **F-8. 불러오기/취합:** 본사가 여러 지사 파일을 한 번에 불러와 합치기
- **F-9. 검증·불일치 표시:** 중복 관리번호, 형식 오류, 지사 간 충돌 데이터 자동 표시
- **F-10. 연 1회 재확인 지원:** 전체 데이터 점검용 요약/이상목록 뷰

## 8. 미확정 / 확정 필요 사항

> 빌드 전에 정해야 하는 항목들.

1. **[중요] 실제 지사명 10개** — 아직 미제공. 현재 PRD는 `지사1 ~ 지사10` 자리표시자. 실제 이름/코드 필요.
2. **지사코드 2자리** — 관리번호 규칙용. 지사명 확정 후 코드 부여 필요.
3. **비용 단위 기준** — 원/천원 단위, 부가세 포함·별도 여부.
4. **담당자 입력 방식** — 자유입력 유지 vs 드롭다운(명단 관리).
5. **교환 파일 형식** — 엑셀(.xlsx) vs 데이터파일(.json) vs 둘 다.
6. **[중요] 데이터 보관 방식** — 단일 HTML 도구에서 입력한 데이터를 세션 간 어떻게 유지할지 (매번 파일로 내보내기 / 브라우저 저장소 활용 등). 빌드 방식에 영향.
7. **권한·구분** — 지사용/본사용 화면을 하나로 합칠지, 분리할지.

## 10. 확정 사항 요약

- 1순위 = 이력관리 / 5년 계획은 후속
- 입력 주체 = 지사 + 본사 둘 다
- 환경 = 사내 인트라넷 사용 가능
- 방식 = A (표준 입력 + 파일 교환 취합 HTML 도구)
- 지사 수 = 10개 (실제 명칭 미확정)
- 데이터 항목 = 6장 표준안 그대로 (추가/삭제 없음)
- 설비종류 목록 = 6.3 기본 보기 그대로
